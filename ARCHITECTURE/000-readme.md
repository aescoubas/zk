# Architectural Decisions

This directory contains records of significant architectural decisions made for the Zettelkasten project. These records (ADRs) help us track the history and context of technical choices.

## Format

We follow a lightweight ADR format:
- **Title**: Short description of the decision.
- **Context**: Why was this decision needed?
- **Decision**: What did we do?
- **Consequences**: What are the trade-offs?

## Guidelines

- Keep this directory up to date.
- Review these documents when onboarding or making major changes.
